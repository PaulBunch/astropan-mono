# Инструменты для создания шрифта

## Inkscape предпочтительнее для этой задачи

**Inkscape** — специализированный SVG-редактор. Его внутренний формат и есть SVG, поэтому то, что ты видишь на экране, напрямую соответствует тому, что получит сборщик шрифта (например, Glyphr Studio). Никаких сюрпризов с экспортом.

Критически важная функция: `Path → Union` и `Path → Break Apart` работают предсказуемо и именно так, как ожидает шрифтовой движок. Для глифов это важнее, чем любые дизайн-фичи.

Inkscape бесплатен, open source, работает офлайн и не требует аккаунта.

**Figma** удобнее для UI-дизайна, но для глифов есть трения: экспортируемый SVG содержит лишние атрибуты и иногда группы вместо плоских путей, что требует ручной чистки перед импортом в Glyphr Studio. Плюс браузерный инструмент с аккаунтом — избыточно для задачи рисования 50 символов.

**Итог**
Для рисования глифов: **Inkscape**.

## Как это работает на практике

**Baseline и Cap height** — это просто горизонтальные направляющие (guides) в Inkscape. Создаёшь их один раз через `Edit → Guides` или перетаскиванием с линейки, и все глифы рисуешь относительно них. Это дисциплина рисования, а не функция инструмента.

**Толщина штриха** — решаешь один раз в начале. Например: основной штрих 125 UPM, тонкий штрих 75 UPM. Записываешь как правило и следуешь ему руками. Inkscape покажет толщину через `Object → Fill and Stroke`, но контролируешь ты сам.

**Оптический центр** — полностью ручная работа. Двигаешь глиф на глаз, сравниваешь рядом с соседними символами. Никакой инструмент это не автоматизирует.

### Практический рабочий процесс

Пример правильной последовательность такая:

```
1. Inkscape: создаёшь документ с размером = 1000×1000px (= 1000 UPM)
2. Inkscape: расставляешь горизонтальные guides:
   - y=0      → baseline
   - y=700    → cap height (стандартное соотношение)
   - y=500    → optical center
   - y=-200   → descender (для символов с хвостом вниз)
3. Inkscape: рисуешь все глифы в одном файле, каждый в своём фрейме
4. Glyphr Studio: импортируешь SVG, расставляешь advance width
5. Glyphr Studio: экспортируешь .woff2
```

### Где FontForge реально нужен

Только если возникнут проблемы с **хинтингом** — это инструкции для растеризатора как отображать глиф на маленьких размерах (8–12px). Для экранного шрифта при размерах 14px+ хинтинг некритичен, и Glyphr Studio справится без FontForge. Если шрифт будет использоваться мелко — тогда да, FontForge для ручного хинтинга незаменим.

## Использования единого SVG‐файла для всех глифов в Glyphr Studio

Один файл для глифов имеет три приимущества: 
- видишь весь набор сразу и оцениваешь согласованность, 
- одни направляющие на весь файл, 
- один файл для импорта в Glyphr Studio.

Glyphr Studio умеет импортировать SVG двумя способами: либо один файл на глиф, либо один файл со всеми глифами где каждый находится в своём именованном слое.

Структура `glyphs.svg` для Glyphr Studio:

```html
<svg>
  <!-- Слой с именем глифа — Glyphr читает атрибут id или label -->
  <g id="U+2609" inkscape:label="☉ Sun">
    <path d="..."/> <!-- контур глифа -->
  </g>

  <g id="U+263D" inkscape:label="☽ Moon">
    <path d="..."/>
  </g>

  <!-- и так далее -->
</svg>
```

Каждый глиф в своём `<g>` с id равным Unicode-кодпоинту. Inkscape управляет слоями именно через `<g>` с атрибутами — это совпадает с тем что ожидает Glyphr Studio.

## Сборка шрифта ptyhon‐скриптом

Для FOSS-проекта с отдельным репозиторием `build.py` — это не просто удобство, это манифест открытости: любой желающий видит как именно SVG превращается в шрифт, может воспроизвести сборку на своей машине, может предложить новый глиф через pull request добавив один файл.

Glyphr Studio как единственный путь сборки создаёт скрытое звено: нужно открыть браузер, загрузить файлы вручную, нажать кнопки. Это не воспроизводимо и не автоматизируемо.

Минимальный `build.py` для этого стека:

```python
# build.py
# Зависимости: pip install fonttools brotli

from fonttools import ...

# 1. Читает все SVG из src/glyphs/
# 2. Сопоставляет имя файла с Unicode-позицией
# 3. Собирает TTF через fonttools
# 4. Оптимизирует в woff2 через pyftsubset
# 5. Кладёт результат в dist/
```

`fonttools` — это тот же инструмент которым пользуется Google Fonts для сборки своих шрифтов. Это индустриальный стандарт для open source шрифтовых проектов.

Итоговый стек:

```
Inkscape (рисование) 
  → src/glyphs/*.svg (хранение, git)
    → build.py + fonttools (сборка)
      → dist/astropan-mono.woff2
```

Glyphr Studio остаётся для визуальной проверки метрик и промежуточного просмотра — но не как единственный путь сборки.
