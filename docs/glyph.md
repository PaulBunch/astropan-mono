# Инструкция по подготовке глифов в Inkscape

Для корректной сборки шрифта каждый символ должен быть представлен в виде **единого замкнутого контура**. «Живые» фигуры (круги, прямоугольники) или пересекающиеся линии компилятор может проигнорировать или обработать с ошибками.

## Правила организации слоев
Для удобства работы и автоматизации рекомендуется разделять «черновик» и «финал»:

* **construction** — рабочий слой. Здесь вы рисуете примитивами, используете обводки (Stroke) и накладываете фигуры друг на друга. Этот слой игнорируется скриптом сборки.
* **glyph** — слой-артефакт. Сюда помещается итоговый результат подготовки. Скрипт `build.py` ищет данные только в этом слое.

---

## Алгоритм подготовки (Check-list)
Если вы закончили отрисовку символа в слое `construction`, выполните следующие шаги для создания финального контура:

1. **Дублирование**: Выделите все объекты в слое `construction` и нажмите `Ctrl + D`.
2. **Оконтуривание** (Object/Stroke to Path):
  * Нажмите `Ctrl + Alt + C`, чтобы превратить толщину линий (Stroke) в контуры.
  * Нажмите `Shift + Ctrl + C`, чтобы превратить примитивы (прямоугольники, эллипсы) в кривые Безье.
3. **Слияние** (Union): Нажмите `Ctrl + +` (Сумма), чтобы сплавить все пересекающиеся части в один объект. Это исключит появление «дырок» и артефактов при масштабировании.
4. **Перенос**: Переместите полученный объект в слой `glyph`.
5. **Очистка** (Optional):
  1. Проверьте узлы: удалите лишние точки вручную.
  2. Не используйте `Ctrl + L` (Simplify) без необходимости, так как это может исказить мелкие детали геометрии.
   
## Технические требования к финальному контуру

* **Тип данных**: Тег `<path>` с атрибутом `d="..."`.
* **Заливка**: Только сплошной цвет (Fill), без градиентов и обводок.
* **Замкнутость**: Контур обязан быть замкнутым.

## Удаление трансформации с объекта path в Inkscape

Проблема заключается в том, что `transform` в SVG — это «неразрушающее» изменение. Координаты внутри атрибута `d` остаются исходными, а видеокарта или рендерер просто пересчитывают их при отображении. Чтобы удалить атрибут без искажений, нужно выполнить операцию **Baking** (запекание) или **Flattening** (сплющивание).

Вот три способа решить эту задачу: от ручного в Inkscape до программного на Python.

### Способ в Inkscape (GUI)

В самом Inkscape нет одной кнопки «Bake Transform», но есть пара приёмов, которые заставляют программу пересчитать координаты:

1.  **Выделите объект** и нажмите `Ctrl+Shift+C` (Object to Path). Иногда этого достаточно.
2.  Если не помогло: **перенесите путь в новую группу**, а затем вынесите его обратно (`Ctrl+G`, затем `Ctrl+U`). При разгруппировке Inkscape часто пытается «втолкнуть» трансформацию группы в координаты пути.

### Программный способ (Python)

Если вы автоматизируете пайплайн сборки шрифта, лучше всего использовать библиотеку **`svgelements`**. Она умеет парсить пути и автоматически применять трансформации к командам (`MoveTo`, `LineTo`, `CurveTo`).

### Пример кода:

```python
from svgelements import SVG, Path, Matrix

def flatten_svg_path(svg_file_path):
    # Загружаем SVG
    svg = SVG.parse(svg_file_path)
    
    for element in svg.elements():
        if isinstance(element, Path):
            # Применяем все накопленные трансформации к координатам 'd'
            element.apply_transform()
            
            # Теперь атрибут transform пуст (Identity Matrix), 
            # а координаты в d пересчитаны.
            print(f"New path data: {element.d()}")
            
            # Удаляем атрибут трансформации
            element.transform.reset()
```

### Почему это важно для шрифтов?

Шрифтовые движки (и формат `.ufo`, и FontForge) обычно не поддерживают вложенные трансформации в описании глифов. Им нужны «чистые» целые или дробные координаты относительно нулевой точки.

### Что выбрать?

  * Если это **единичный глиф**, который вы рисуете руками — используйте трюк с группировкой/разгруппировкой в Inkscape.
  * Если это **часть пайплайна**, интегрируйте `svgelements`. Это избавит от необходимости вручную «чистить» каждый файл перед сборкой.
