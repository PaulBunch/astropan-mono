# Направляющие

`guidelines.svg`, это именно подложка — шаблон с направляющими линиями, который ты открываешь как фоновый слой или отдельный документ рядом с `glyph.svg`.

Вот что в нём должно быть (конкретные значения уточняются в ходе разработки):

```
guidelines.svg
│
├── Слой "grid" (заблокирован, не редактируется)
│   ├── Вертикальная сетка: линии каждые 100 UPM
│   └── Горизонтальная сетка: линии каждые 100 UPM
│
├── Слой "metrics" (заблокирован)
│   ├── y=800  → Ascender (красная линия)
│   ├── y=700  → Cap height (синяя линия)
│   ├── y=500  → Optical center (зелёная пунктирная)
│   ├── y=0    → Baseline (чёрная, толстая)
│   └── y=-200 → Descender (красная линия)
│
├── Слой "advance-width" (заблокирован)
│   ├── Прямоугольник 1000×1000 — граница em square
│   ├── Вертикаль x=50  → левый sidebearing
│   └── Вертикаль x=950 → правый sidebearing
│
└── Слой "stroke-reference" (заблокирован)
    ├── Прямоугольник шириной 100 UPM → эталон основного штриха
    └── Прямоугольник шириной 60 UPM  → эталон тонкого штриха
```

Два способа использования. 
- Первый — копируешь `guidelines.svg` для каждого нового глифа, рисуешь поверх направляющих, сохраняешь под именем символа. 
- Второй — открываешь `guidelines.svg` как фон в Inkscape через `File → Import` с опцией "Link" (не embed), тогда при изменении шаблона все файлы обновятся автоматически.

Для одного файла `glyph.svg` с символом: помещаешь слой guidelines в самый низ, блокируешь, и рисуешь глиф поверх в отдельном слое. Направляющие тянутся через весь файл горизонтально — это и есть главная ценность, видишь все символы на одних и тех же линиях одновременно.

# Отдельный файл на глиф + сборка через скрипт

Git-история становится читаемой. `modified: src/glyphs/saturn.svg` говорит всё сразу. `modified: src/glyphs.svg` не говорит ничего.

Конфликты слияния локализованы. Если два контрибьютора правят разные глифы — конфликта нет вообще.

`<use>` в Inkscape для сборки предварительного просмотра — хорошая идея, но с оговоркой: Inkscape реализует `<use>` через `xlink:href` который deprecated в SVG 2.0, и при экспорте может вести себя непредсказуемо. Надёжнее собирать строку скриптом в отдельный `preview.svg` который не является рабочим файлом.

Итоговая структура:

```
src/
├── glyphs/
│   ├── sun.svg        # U+2609
│   ├── moon.svg       # U+263D
│   ├── saturn.svg     # U+2644
│   └── ...
├── guidelines.svg     # Шаблон направляющих
└── preview.svg        # Генерируется скриптом, в .gitignore
```
