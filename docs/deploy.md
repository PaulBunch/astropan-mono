# Как сделать specimen доступным на GitHub

`dist/` в `.gitignore` — правильное решение для бинарных артефактов сборки. Для specimen есть два чистых подхода:

**Вариант A — GitHub Actions** автоматически собирает шрифт и деплоит `docs/` на GitHub Pages при каждом пуше. Это идеальное решение: источник истины остаётся в `src/`, а Pages всегда показывает актуальный шрифт.

**Вариант B — отдельная ветка `gh-pages`** куда вручную или через скрипт кладётся собранный артефакт. Менее автоматично, но проще в настройке.

Для проекта на этой стадии рекомендуется Вариант A. Вот минимальный workflow:Файл кладётся в `.github/workflows/deploy.yml`. После этого в настройках репозитория: **Settings → Pages → Source → GitHub Actions**.

Важный момент по структуре: `specimen.html` сейчас лежит в `docs/` и обращается к шрифту через `/dist/`. На GitHub Pages нужно тоже запускать сервер из корня, чтобы не держать два разных пути для локальной работы и Pages.

```javascript
const FONT_URL = '/dist/astropan-mono.ttf'; // работает и локально (с сервером), и на Pages
```

Локально запускать через `python -m http.server` из корня проекта.
