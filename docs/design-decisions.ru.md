# Design Decisions: AstroPan Font

Этот документ фиксирует ключевые архитектурные и эстетические решения проекта. Основная цель — создание набора астрологических символов, оптимизированных для читаемости на сверхмалых кеглях (**10px**) в среде разработчика.

## 1. Фундаментальные принципы
* **Решительность формы вместо детализации:** На 10px любая деталь меньше 1px (100 UPM) превращается в шум. Приоритет отдается сохранению внутренних просветов (counter-space).
* **Оптическая согласованность:** Шрифт проектируется как визуальное дополнение к **JetBrains Mono**, наследуя его «технический» характер и вертикальные метрики.
* **Приоритет ясности над системностью:** Выбор ширины площадки продиктован не математической кратностью сетке терминала, а физической разборчивостью знака.

---

## 2. Геометрические метрики

| Параметр          | Значение    | Обоснование                                                                                              |
| :---------------- | :---------- | :------------------------------------------------------------------------------------------------------- |
| **UPM**           | 1000        | Промышленный стандарт для точного контроля координат.                                                    |
| **Advance Width** | **800 UPM** | **Финальное решение.** Обеспечивает 8px ширины на 10px кеглях. Баланс между плотностью и разборчивостью. |
| **Cap Height**    | 730 UPM     | Согласовано с высотой маюскулов JetBrains Mono.                                                          |
| **x-Height**      | 550 UPM     | Согласовано со строчными JetBrains Mono.                                                                 |
| **Sidebearings**  | 50–100 UPM  | Минимальный «зазор безопасности» для предотвращения слипания знаков.                                     |

---

## 3. Язык форм и штрих

Шрифт разделен на две функциональные группы с разным подходом к отрисовке:

### 3.1. Планеты и Аспекты (Геометрическая группа)
* **Тип штриха:** Моноширинный (Constant weight).
* **Толщина:** Основной штрих — **80 UPM**, дополнительные элементы — **50 UPM**.
* **Концепт:** Использование круга как базового элемента для всех планетарных глифов.
* **Правило зазоров:** Минимальное расстояние между параллельными штрихами — **100 UPM** (1px).

### 3.2. Знаки Зодиака (Каллиграфическая группа)
* **Тип штриха:** Динамический (Dynamic stroke) с ручной компенсацией толщины.
* **Цель:** Сохранение традиционной узнаваемости знаков при высокой плотности элементов (особенно для Девы ♍ и Скорпиона ♏).
* **Скругления:** Агрессивные радиусы (**20+ UPM**) для предотвращения появления «клякс» при растеризации углов.

---

## 4. Правила растеризации (Grid-Fitting)
* **Привязка к сетке:** Все узлы и границы штрихов должны тяготеть к мажорному шагу в **100 UPM**.
* **Оптические вылеты (Overshoots):** Скругленные элементы (круги планет, дуги Овна) выходят за базовую линию и Cap Height на **10–20 UPM** для компенсации иллюзии уменьшения круга.
* **Центровка:** Все глифы центрируются строго геометрически относительно площадки в 800 UPM.

---

## 5. Отвергнутые альтернативы
* **Advance Width 600 UPM:** Отвергнуто из-за невозможности сохранить структуру сложных знаков (Плутон, Дева) без падения толщины штриха ниже критических 50 UPM.
* **Advance Width 900 UPM (1.5x от 600):** Несмотря на стандартную кратность, этот вариант был отвергнут после тестов на 10px (144 DPI). Вертикально симметричные глифы с деликатными элементами (например, Плутон) теряют чёткость.
* **Double-width (1200 UPM):** Отвергнуто как избыточно широкое, создающее слишком большие визуальные разрывы в текстовом полотне.

---

## 6. Версионирование

В шрифте используется формат версионировния `MAJOR.MINOR`, где minor записывается тремя цифрами. Для шрифтов устоявшаяся практика:

| Цифра            | Когда меняется                                                       | Пример            |
| :--------------- | :------------------------------------------------------------------- | :---------------- |
| Целая (1.xxx)    | Новая смысловая группа глифов, смена метрик, несовместимые изменения | `1.000` → `2.000` |
| Десятые (x.1xx)  | Новые глифы в рамках существующей системы                            | `1.000` → `1.100` |
| Сотые (x.x1x)    | Исправления контуров, координат, дизайнерские правки                 | `1.000` → `1.010` |
| Тысячные (x.xx1) | Метаданные, технические исправления без изменения форм               | `1.000` → `1.001` |
