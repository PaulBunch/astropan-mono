# AstroPan Mono — Техническое задание на шрифт

## 1. Обзор проекта
**Название:** AstroPan Mono
**Тип:** Моноширинный иконочный шрифт
**Назначение:** Отображение астрологических символов в интерфейсе приложения AstroPan и других совместимых проектов
**Лицензия:** SIL Open Font License 1.1 (OFL)
**Формат дистрибуции:** `.woff2` (основной), `.ttf` (архивный)

---

## 2. Цели и ограничения
* **Главная цель:** Создать согласованный набор символов, сохраняющий разборчивость на малых размерах (**10px**) при плотности экрана 96 dpi.
* **Единый язык:** Одинаковая толщина штриха, радиусы скруглений и оптическая плотность для всех глифов.
* **Совместимость:** Пропорции должны позволять бесшовное использование шрифта совместно с **JetBrains Mono**.

---

## 3. Технические характеристики

### 3.1 Метрики шрифта

**Глиф**
| Параметр                      | Значение                    | Обоснование                  |
| :---------------------------- | :-------------------------- | :--------------------------- |
| Units Per Em (UPM)            | 1000                        |                              |
| Ascender                      | 800                         |                              |
| Descender                     | -200                        |                              |
| Cap height                    | 730                         | Согласовано с JetBrains Mono |
| x-height                      | 550                         | Согласовано с JetBrains Mono |
| Baseline                      | 0                           |                              |
| Advance width                 | смотри раздел 3.2           |                              |
| Боковые отступы (sidebearing) | 50–100 UPM с каждой стороны |                              |

**Штрих**
| Параметр                  | Значение | Обоснование                              |
| :------------------------ | :------- | :--------------------------------------- |
| Толщина основного штриха  | 100 UPM  | При 10px = ровно 1px (попадание в сетку) |
| Толщина тонкого штриха    | 60 UPM   | Не менее 0.6 от основного                |
| Мин. зазор между штрихами | 100 UPM  | Не меньше толщины основного штриха       |
| Скругление углов          | 20 UPM   | Смягчение «клякс» при растеризации       |

**Сетка**
- Все узлы привязаны к минорному шагу сетки: 10 UPM
- Ключевые границы штрихов (outline) привязываются к мажорному шагу сетки: 100 UPM (для попадания в физические пиксели при 10px)

### 3.2. Адаптивная моноширинность
Проект использует систему кратности площадок для сохранения читаемости сложных глифов:

1. **Narrow (Базовый модуль):** Advance width **600 UPM**. Для большинства глифов.
2. **Wide (Расширенный модуль):** Advance width **900 или 1200 UPM**. Применяется, если внутренние просветы символа при ширине 600 UPM становятся менее 100 UPM.
3. **Правило групповой преемственности:** Если один глиф смысловой группы (например, Дева в знаках зодиака) требует Wide-модуля, вся группа переводится на этот модуль для сохранения визуального ритма.
4. **Оптические вылеты:** Допускается вынос скругленных элементов за пределы площадки (overshoot) до 10 UPM.
5. **Критерий выбора модуля:** Если внутренние зазоры глифа в базовом модуле становятся меньше 100 UPM, выбирается расширенный модуль. Приоритет отдается сохранению толщины основного штриха 100 UPM над компактностью по ширине.

**Эталоны плотности**
- Для знаков: Дева
- Для планет: Плутон

Стандартных Unicode-точек для домов не существует.
Использовать зону Private Use Area (PUA): U+E000–U+E00B.

| Код    | Название |
| :----- | :------- |
| U+E000 | Дом I    |
| U+E001 | Дом II   |
| ...    | ...      |
| U+E00B | Дом XII  |

Для удобства использования в Web создать CSS-карту классов (`.icon-house-1`).

### 6. Порядок разработки

**Фаза 0 — Система (до рисования)**
- [ ] Создать `guidelines.svg`: em square 1000×1000, направляющие: baseline (0), cap height (730), x-height (550), geometric center (500), descender (-200)
- [ ] Определить optical center визуально и зафиксировать значение
- [ ] Нарисовать тестовый прямоугольник с толщиной штриха 100 UPM, проверить читаемость на 10px в браузере
- [ ] Зафиксировать финальные значения метрик в этом документе

**Фаза 1 — Эталоны (5 символов)**
Цель: отработать систему на крайних случаях сложности.
- [ ] ☌ Соединение — простейший
- [ ] □ Квадратура — угловой
- [ ] ♈ Овен — средний
- [ ] ♄ Сатурн — сложный составной
- [ ] ⯓ Плутон — максимальная насыщенность

Критерий перехода к Фазе 2: все 5 символов однозначно читаются при `font-size: 10px` в браузере Chrome и Firefox.

**Фаза 2 — Полный набор (39 символов)**
Порядок: аспекты → планеты → знаки зодиака → дома.
Аспекты первыми: геометрически простейшие (круг, квадрат, треугольник), задают язык форм — толщину штриха, скругления — для всего набора.
Дома последними: нет Unicode-референса, форма определяется свободно после того как язык форм устоялся.

**Фаза 3 — Хинтинг и оптимизация**
- [ ] Хинтинг в FontForge если есть артефакты на 10px
- [ ] `pyftsubset` → финальный `.woff2`
- [ ] Проверить все глифы на 10, 12, 14, 16px в `specimen.html`

## 7. Критерии качества
Шрифт считается готовым к релизу если:

1. **Читаемость:** 100% распознаваемость всех 39 символов при кегле 10px в Chrome и Firefox на дисплее 96 dpi
2. **Геометрия:** Отклонение толщины штриха от 100 UPM не более ±5 UPM
3. **Просветы:** Внутренние пустые пространства (counters) ≥ 50 UPM (мнимум 0.5px)
4. **Центровка:** Все глифы оптически сбалансированы относительно центра площадки
5. **Вес:** Размер файла `.woff2` менее 20 КБ
6. **Производительность:** Отсутствие визуальных артефактов FOUT (flash of unstyled text) при `font-display: block`
