# Подготовка прототипов

## Три способа, от простого к сложному

**1. DevTools → Inspector (самый простой)**

Открываешь DevTools (F12), вкладка Elements. Находишь нужный `<svg>` или `<path>` элемент, правый клик → "Copy" → "Copy outerHTML". Вставляешь в Inkscape через `File → Import` или просто в текстовый файл с расширением `.svg`.

**2. DevTools → Network (если символы загружаются как файл)**

Вкладка Network, фильтр по `svg` или `font`. Если сайт грузит SVG-спрайт или шрифтовой файл — он будет виден здесь. Правый клик на запросе → "Save as".

**3. DevTools → Sources (если символы инлайновые)**

Если `<svg>` зашит прямо в HTML, ищи в Sources → найди основной HTML-файл → Ctrl+F по `<svg` или `<symbol`.

## Что делать с найденным SVG в Inkscape

Открываешь файл, затем обязательно:

```
Path → Object to Path   (конвертируешь всё в пути)
Path → Union            (объединяешь в один контур)
File → Clean Up Document (убираешь мусор)
```

## Важный момент про авторские права

Технически скопировать можно почти всё. Юридически — зависит от лицензии. Астрологические символы как таковые не защищены авторским правом (это стандартные Unicode-символы), но конкретная художественная реализация глифа может быть. Если берёшь за основу чужой глиф для коммерческого проекта — лучше использовать его как референс для перерисовки в Inkscape, а не копировать напрямую.

## Эталонные символы JetBrains Mono как прототипы

| Эталон                         | Прототип в JetBrains Mono | Что изучать                                                                               |
| :----------------------------- | :------------------------ | :---------------------------------------------------------------------------------------- |
| ☌ Соединение (круг)            | **O** заглавная           | Как построен идеальный круг на сетке, толщина овала, боковые отступы                      |
| □ Квадратура (квадрат)         | **[** или **0**           | Острые углы и их скругление, толщина горизонтальных vs вертикальных штрихов               |
| ♈ Овен (два полукруга)         | **ω** или **∞**           | Сопряжение двух кривых в одной точке, симметрия                                           |
| ♄ Сатурн (крест + полукруг)    | **ħ** или **f**           | Пересечение вертикального и горизонтального штриха, как штрихи разной толщины встречаются |
| ♇ Плутон (круг+крест+полукруг) | **φ** или **⊕**           | Композиция нескольких примитивов в ограниченном пространстве, иерархия элементов          |


Скачай исходники JetBrains Mono с GitHub — там есть `.glyphs` файл, который открывается в Glyphr Studio. Можно буквально измерить координаты узлов и толщины штрихов.

## Работа с SVG-файлами

**Что оставить из атрибутов:** только `viewBox` и `<path d="...">`. Всё остальное — мусор для референсного файла.

Минимальная чистая форма:

```html
<svg viewBox="0 0 24 24" xmlns="http://www.w3.org/2000/svg">
  <path d="M12 4C16.41 4 20 7.59..."/>
</svg>
```

Убрать: `fill`, `stroke`, `width`, `height`, `x`, `y`, `class`, `id`, `data-*` атрибуты, `fill-rule` можно оставить если он влияет на форму (evenodd vs nonzero — это про дырки внутри контура).

**Отдельные файлы или один:** для референсов — отдельный файл на каждый глиф. Это даёт гибкость переставлять их в строке.

**Размер холста референсной строки:** приведи все глифы к единому viewBox перед сборкой. Оптимально `0 0 1000 1000` — совпадает с em square. Масштабирование через `transform="scale(41.67)"` (1000/24 = 41.67 для глифов с viewBox 24×24).

**Точная расстановка в строке:** используй `<use>` с координатами:

```html
<!-- Референсная строка: файл reference-row.svg -->
<svg viewBox="0 0 12000 1000" xmlns="http://www.w3.org/2000/svg">

  <!-- Направляющие (общие для всей строки) -->
  <line x1="0" y1="800" x2="12000" y2="800" stroke="red" stroke-width="2"/>
  <line x1="0" y1="700" x2="12000" y2="700" stroke="blue" stroke-width="2"/>
  <line x1="0" y1="0"   x2="12000" y2="0"   stroke="black" stroke-width="3"/>

  <!-- Глиф 1: позиция x=0, трансформация scale + translate -->
  <g transform="translate(0, 0) scale(41.67)">
    <path d="M12 4C16.41..."/> <!-- Sun ☉ -->
  </g>

  <!-- Глиф 2: сдвиг на 1000 UPM (= один advance width) -->
  <g transform="translate(1000, 0) scale(41.67)">
    <path d="M9.00016 22..."/> <!-- Moon ☽ -->
  </g>

  <!-- Глиф 3 -->
  <g transform="translate(2000, 0) scale(41.67)">
    <path d="..."/> <!-- Mercury ☿ -->
  </g>

</svg>
```

Шаг между глифами всегда кратен 1000 — это и есть advance width твоего шрифта. Так координаты предсказуемы без вычислений.

**Переставлять глифы:** меняешь только значение `translate(X, 0)` — первый параметр. Это быстро через Find & Replace в любом текстовом редакторе. Или держишь глифы в отдельных файлах и собираешь строку скриптом:

```python
# assemble_reference.py
glyphs = ["sun.svg", "moon.svg", "mercury.svg", "venus.svg"]
# скрипт читает каждый файл, извлекает <path>, 
# вставляет в нужную позицию x = index * 1000
```

Это займёт 20 строк Python и даст полную гибкость перекомпоновки без ручного редактирования координат.

**Итоговая файловая структура референсов:**

```
src/
├── references/          # Не коммитится (.gitignore)
│   ├── single/          # Отдельные глифы
│   │   ├── sun.svg
│   │   ├── moon.svg
│   │   └── ...
│   ├── rows/            # Собранные строки
│   │   ├── planets-ref.svg
│   │   ├── signs-ref.svg
│   │   └── aspects-ref.svg
│   └── assemble.py      # Скрипт сборки строк
├── guidelines.svg
└── glyphs.svg
```

`references/` — это твои личные рабочие материалы, не часть публичного проекта.
